## Описание
JS разработчик обратился к Вам за помощью с написанием автоматизации
развертывания приложения на локальной машине для тестирования

## Структура

`frontend` - Клиентская часть приложения (написана на JS + React). 
Открытый порт - _3002_. Node JS version = `14.16.1`

`backend` - Серверная часть приложения (NestJS). 
Открытый порт - _3001_. Node JS version - `14.16.1`

Внутри каждого модуля есть описание, как его можно развернуть

Первый запуск приложения со сборкой и загрузкой зависимостей очень долгий, даже запущенный контейнер очень медленно стартует

## Требования к Dockerfile
1) Переменные окружения (env) должны иметь возможность
   переопределяться из командной строки, следовательно,
   задаваться как аргументы. В случае отсутсвия значения -
   принимать значение по умолчанию (брать из соответсвующего 
   Readme файла) - *2 балла*
2) Добавить к контейнерам метадату в виде меток - *1 балл*:
   1) author - your nickname
   2) version - <придумайте любую>
   3) development - dev (указание, что этот образ для стенда разработки)
   4) description - <любое короткое описание>
3) Указан открытый порт(ы), если такие есть - *1 балл*
4) Указаны монтируемые volumes, если есть - *1 балл*
5) Запуск приложения внутри контейнера происходит не под пользователем root (ДОКАЗАТЬ, приложив фотографию вывода команды `ps aux` внутри контейнера, что запуск произведен не под UID 1 (root)) - *1 балл без фото, 3 балла с фото*
6) Добавлен healthcheck для контейнера - *1 балл за сам хэлсчек, 2 балла - если настроите с помощью параметров*
7) Добавлен dockerignore файл - *1 балл*

## Задание 
1) Создать Dockerfile для сборки образа бэкенда. - *2 балла*
2) Создать Dockerfile для сборки фронтенда - *2 балла*
3) Создать Dockerfile для развертывания БД: 
   1) MySQL 8.0
   2) port 3306:3306
   3) DB name: tradeoffer_db
   4) DB password: root
   5) Либо создать Docker Volume для БД, либо примонтировать папку
   Итоговый результат - при перезапуске БД данные в ней должны сохраняться - *2 балла за БД без Volume, 3 балла за volume*
4) Создать Dockerfile для развертывания `adminer:latest` - 
админ-панели для БД. - *2 балла*
Переменные окружения:
   1) ADMINER_DEFAULT_DB_DRIVER: mysql
   2) ADMINER_DEFAULT_SERVER: mysql_db
   3) ADMINER_PLUGINS: tables-filter tinymce
5) Написать файл `deploy_app.sh`, в котором перечислить все команды для запуска приложения - *2 балла*

Итоговый результат - запущенное приложение на 3002 порту! Следовать требованиям необязательно,
но за их выполнение можно получить дополнительные баллы
